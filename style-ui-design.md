# Style UI 設計メモ

## 結論

このツールの本体は `ブログタイトルから画像生成プロンプトを作ること`。
`Style` はそのための補助機能であり、通常利用では深く触らなくても使えることを優先する。

## ツールの役割

- タイトルを入力する
- 画像サイズを選ぶ
- Style を選ぶ
- プロンプトを生成する

ここが最短導線。

## Style 機能の役割

- 標準Styleを選んでそのまま使える
- 必要なら中身を見られる
- 必要なら My Style を作れる
- My Style は更新できる
- My Style は削除できる

## 分けるべき責務

Style 周りでは次を混ぜない。

- `使う`
- `見る`
- `作る`
- `更新する`
- `削除する`

## 基本フロー

### 標準利用

1. Style を選ぶ
2. そのまま生成する

### 詳細確認

1. Style を選ぶ
2. `中身を見る` を押す
3. パネルで中身を確認する
4. 閉じる

### My Style 作成

1. 標準Styleを選ぶ
2. `中身を見る` を押す
3. 内容を調整する
4. Style名を入れる
5. `My Style を作る` を押す

### My Style 更新

1. My Style を選ぶ
2. `中身を見る` を押す
3. 内容を調整する
4. `更新する` を押す

### My Style 削除

1. My Style を選ぶ
2. `中身を見る` を押す
3. `削除する` を押す

## UI 原則

- 普段はシンプルに保つ
- 編集パネルは必要な時だけ開く
- 開いた時は「今どのStyleを見ているか」が明確である
- 標準Styleと My Style の違いが一目で分かる
- 保存前の内容と、実際に生成に使われる内容を混同させない

## 状態設計

必要な状態は最小限に絞る。

- `selectedStyleKey`
- `editorOpen`
- `customStyles`

必要なら派生値として扱う。

- `isCustomSelected`
- `selectedStyle`

## 実装ルール

- 生成は常に `selectedStyleKey` に紐づく保存済みStyleを使う
- 編集中のテキストは、保存するまでは生成に反映しない
- `中身を見る` は開閉だけ担当する
- `My Style を作る` / `更新する` は保存だけ担当する
- `削除する` は対象が現在選択中のときだけ fallback する

## この設計で避けること

- 開くボタンと保存ボタンを兼用しない
- 見るUIと編集UIを曖昧にしない
- 標準Styleを選んだだけで勝手に保存モードにしない
- 削除時に無関係な選択状態を壊さない

## 次の実装方針

1. `index.html` の Style UI をこの設計に合わせて整理する
2. `js/ui.js` を状態管理中心に作り直す
3. `generate()` は保存済みStyleのみを参照するよう固定する
